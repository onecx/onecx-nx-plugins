= Configure Permissions

In order to restrict users with certain roles from performing certain actions, permissions need to be introduced.
To configure the permissions, the following actions must be taken. + 
Refer to the {permissions_inangular_link} for more information.


[#action-11]
== ACTION S11

Configure permissions::
Adapt the permissions in the `+helm/values.yaml+` file.

[discrete]
=== Template: Permissions

[horizontal]
Directory:: `<root-of-ui-app>/helm/`
File::      `values.yaml`

.ACTION S11 in `+values.yaml+`
[%collapsible]
====
[source, javascript]
----
    # Permission
    permission:
      enabled: true
      spec:
        permissions:
        # ACTION S11 Add entity permissions
----
====


[discrete]
=== Example
Configure permissions for the `Book` entity by adding the following snippet to the `values.yaml` file.

[horizontal]
Directory:: `bookstore/helm/`
File::      `values.yaml`

.ACTION S11 in `+values.yaml+`
[%collapsible]
====
[source, javascript]
----
    permission:
      enabled: true
      spec:
        permissions:
          BOOK:
            CREATE: Create Book item
            EDIT: Edit Book item
            DELETE: Delete Book item
            SEARCH: Search Book items
            VIEW: View Book item details
            IMPORT: Import Book items
            EXPORT: Export Book items
            BACK: Navigate back in details of Book item
----
====
